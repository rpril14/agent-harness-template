# Execution Plans

An execution plan is a self-contained document that lets an agent complete complex work without relying on hidden context.

Use a plan for:

- Complex features.
- Significant refactors.
- Architecture changes.
- Data model or migration work.
- Security-sensitive changes.
- Multi-step tasks where progress should survive context loss.

Plan files belong in `docs/plans/active/` while in progress and `docs/plans/completed/` after completion.

Use `templates/execution-plan.md` as the plan format.
