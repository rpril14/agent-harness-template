# Glossary

Use this file to keep domain language stable across product docs, code, UI, APIs, database names, and tests.

## Terms

### User

A person who uses the product.

### Account

The ownership, billing, or administrative container for one or more users.

### Workspace

A collaborative area or logical container where users do work.

### Project

A user-created unit of work inside the product.

## Naming Rules

- Use glossary terms consistently in code, UI labels, tests, and documentation.
- Add a term before introducing a new domain concept.
- If a term changes meaning, update affected docs and code references.
